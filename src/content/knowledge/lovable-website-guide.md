Erfahre, wie Du mit Lovable professionelle Websites erstellst, die nicht nur gut aussehen, sondern auch Ergebnisse liefern. Hier ist das komplette Framework für 2026.

## 🏗️ Layout & Hero Section

Wähle ein bewährtes Hero-Layout. Deine Hero-Section muss innerhalb von 5 Sekunden drei Fragen beantworten: Was ist das? Für wen ist es? Warum ist es wichtig?

### Bewährte Layout-Optionen:
*   **A:** Große Headline + Produkt-Screenshot (rechts)
*   **B:** Zentrierte Headline + 3 Bullets + Screenshot darunter
*   **C:** Split: Value Proposition links, Proof (Social Proof) rechts

**Lovable Prompt für Layout B:**
> Create a hero section using layout option B: Centered headline, 3 concise value bullets, and a product screenshot below. Add a primary CTA button. Keep spacing generous and the layout clean and premium.

## 📈 Die konvertierende Sektions-Reihenfolge

Ein erfolgreicher Flow führt den Nutzer logisch durch das Angebot:

1.  **Hero:** Value Proposition + CTA
2.  **Social Proof:** Glaubwürdigkeit aufbauen (Logos, Partner)
3.  **Problem → Lösung:** Erkläre den Kern des Problems und Deine Lösung
4.  **Feature → Outcome Mapping:** Verknüpfe Funktionen mit konkreten Ergebnissen
5.  **Visual Demo:** Zeige das Produkt in Aktion
6.  **Secondary Proof:** Testimonials und detaillierte Kundenmeinungen
7.  **Final CTA:** Der letzte Abschluss-Button

## 🎨 Visual Design: Das Premium-Gefühl

Premium-Seiten wirken ruhig und aufgeräumt. Der größte Fehler bei AI-generierten Seiten ist zu wenig Platz.

### Spacing & Rhythmus
*   **Atemraum:** Erhöhe den vertikalen Abstand zwischen Sektionen.
*   **Wechsel-Rhythmus:** Lange Seiten wirken leichter, wenn sich das visuelle Muster abwechselt (z.B. Light → Dark → Light oder Bild links → Bild rechts).

### Kontrast & Typografie
Du brauchst keine Verläufe überall. Nutze White Space, subtile Rahmen und weiche Schatten. Halte die Typografie einfach: **1 Schriftart, 3 Größen, 2 Gewichte.**

**Design-Prompt:**
> Use white space, subtle borders, and light/dark section contrast. Avoid heavy gradients and visual noise. Use a single font family and limit typography to three sizes and two weights.

## 📱 Mobile & Responsive Design

Die meisten AI-Seiten scheitern auf dem Smartphone, weil Breakpoints nicht definiert sind.

*   **Golden Breakpoints:** Desktop (≥ 1440px), Tablet (≥ 1024px), Mobile Landscape (≤ 768px), Mobile (≤ 375px).
*   **Mobile Regeln:** Kürze die Sektionen, weniger Text, mehr Spacing, CTAs früher im Scroll-Flow anzeigen.

**Mobile-Optimierung Prompt:**
> Make the layout fully responsive. Stack elements below 768px. Hide secondary elements on mobile. Use full-width CTA buttons below 480px. Ensure large tap targets.

## 🎯 Conversion Hacks

Bevor Du designst, kläre die Psychologie: Warum melden sich Nutzer an? Was hindert sie daran? Design ohne diese Klarheit ist nur Dekoration.

*   **Social Proof:** Testimonials sind stärker als Werbetexte. Nutze echte Zitate, Logos und Badges.
*   **Fokus:** Jede Seite sollte nur **eine Aktion** pushen (z.B. "Sign Up").
*   **Spezifische CTAs:** "Erstelle Deinen ersten Bericht" konvertiert besser als ein generisches "Get Started".

## 🔄 Iteration & Testing

Sobald die Struktur steht, teste systematisch:
1.  Headlines
2.  CTA-Wording
3.  Button-Platzierung
4.  Reihenfolge der Sektionen

Lovable macht Iteration trivial: **Duplizieren → Anpassen → Shiffen → Verfeinern.**
