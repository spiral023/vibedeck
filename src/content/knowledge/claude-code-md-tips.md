Die Datei `CLAUDE.md` ist das Gehirn Deines Projekts, wenn Du mit Claude Code arbeitest. Mit den richtigen Regeln kannst Du die Zuverlässigkeit des Agenten massiv steigern. Hier sind fünf bewährte Tipps, die Du direkt übernehmen kannst.

## 1. Erst planen, dann coden

Zwinge Claude dazu, zuerst nachzudenken. Das verhindert unnötige Code-Änderungen und stellt sicher, dass der Agent Dein Ziel verstanden hat.

> "Bevor Du Code schreibst, beschreibe Deinen Ansatz und warte auf meine Freigabe. Stelle immer klärende Fragen, bevor Du beginnst, falls die Anforderungen unklar sind."

## 2. Aufgaben kleinteilig halten

Große Änderungen an vielen Dateien führen oft zu Fehlern oder Orientierungslosigkeit beim Agenten.

> "Falls eine Aufgabe Änderungen an mehr als 3 Dateien erfordert, stoppe und zerlege sie zuerst in kleinere Teilaufgaben."

## 3. Risiken benennen und Tests vorschlagen

Nachdem Claude eine Änderung vorgenommen hat, sollte der Agent selbstkritisch prüfen, was schiefgehen könnte.

> "Liste nach dem Schreiben von Code auf, was kaputtgehen könnte, und schlage Tests vor, um diese Szenarien abzudecken."

## 4. Test-Driven Bug Fixing

Bugs sollten niemals "einfach so" gefixt werden. Ein reproduzierbarer Test ist die beste Basis für einen sauberen Fix.

> "Wenn ein Bug auftritt, schreibe zuerst einen Test, der den Fehler reproduziert. Fixe den Bug erst danach, bis der Test erfolgreich durchläuft."

## 5. Aus Fehlern lernen (Self-Improving)

Jede Korrektur, die Du manuell vornehmen musst, ist eine Chance, die `CLAUDE.md` zu verbessern.

> "Jedes Mal, wenn ich Dich korrigieren muss, füge eine neue Regel zur `CLAUDE.md` hinzu, damit dieser Fehler in Zukunft nicht mehr passiert."

---

Diese Regeln helfen dabei, Claude Code von einem reinen "Code-Schreiber" zu einem mitdenkenden Partner zu machen. Kopiere sie in Deine `CLAUDE.md` und passe sie an Deinen Workflow an.
